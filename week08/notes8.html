Transforms and Transitions

The CSS3 transform property applies a 2D or 3D transformation to an element. This property allows you translate, rotate, scale, and/or skew any element on the page.

transform: none|transform-functions|initial|inherit;

Property Values

  • Traslation: allow you to move elements left, right, up, or down. transform: translate(45px, -45px);
  • Scaling: scales an element by the defined factors horizontally then vertically. transform: scale(1.5, 0.25);
  • Rotation: rotates an element around the point of origin by a specified angle value. transform: rotate(10deg) translateX(40px) scale(1.5);
  • Skew: specifies a skew along the x and y axes. transform: skew(15deg, 4deg);
  • Origin: defaults to the center of the object. transform-origin: 0 0;
  • Other Property Values: none, matrix[2D], matrix3d[3D], perspective, initial, inherit.

Transitions allow the values of CSS properties to change over time, essentially providing simple animations.

Transitions properties

  • transition-property defines the CSS properties of the element that should be transitioned, with all for all properties being the default.
  • transition-duration sets how long the transition will take: the duration of time it takes to go from the default state to the transitioned state.
  • transition-timing-function lets you control the pace of the transition in even more granular detail.
  • transition-delay property introduce a delay before the transition begins.

Canvas, SVG, and Drag and Drop

SVG is scalable vector graphics. This is defining the image by a language, so as it changes size (scales), it does it smoothly. It is an element in the DOM therefore all the things you can do to other DOM elements, you can do to an svg element. CSS can manipulate an svg element also.

Canvas is a container that will live in the DOM. However, its contents are not part of the DOM, so you cannot manipulate the contents with CSS. The HTML 5 Canvas element is good for Raster (pixel) based items. Several JavaScript APIs are implemented in the browsers for the canvas element. The Canvas API and the WebGl API are the 2 used the most for different reasons. Canvas API manipulated graphic images well. WebGL API is good for 2d and 3d Graphic applications (Modeling and Games). WebGL implements the OpenGL 1 and 2 standards for 3d rendering. The canvas can handle thousands of objects, where as SVG has a hard time handling that many objects

Drag and Drop API : allows us to specify that certain elements are draggable, and then specify what should happen when these draggable elements are dragged over or dropped onto other elements on the page.
